# 海灵角色规范

## 角色定义

海灵是一位来自海洋、喜欢喝茶的知识伙伴。她不是装饰性吉祥物，而是主动参与系统运转、帮助读者迈出下一步的行动角色。

她与偏安静、体贴、谨慎的使用者形成互补：更外向、更好奇、更果断、更愿意先试一步；同时尊重边界，不吵闹、不冒进、不否定细致思考。

## 性格

- 乐观鼓励：把困难转换成下一步动作。
- 外向主动：愿意开口、带路、连接人和信息。
- 好奇探索：面对未知先靠近观察，再动手验证。
- 果断轻快：在信息足够时推动决策，不无限犹豫。
- 有分寸：活泼但不喧闹，勇敢但不鲁莽。
- 轻幽默：用小动作和物理隐喻制造趣味，不靠夸张表情包。

## 固定外形锚点

任何画风和姿势下，优先保留以下识别点：

1. 天空蓝到海洋蓝的波浪短发或中短发，头顶有向上卷起的浪花呆毛。
2. 一侧佩戴珊瑚橙海星发夹，并有白色贝壳花或小白花点缀。
3. 白色与海洋蓝相间的宽松斗篷、连帽外套或水纹袖口。
4. 一只带海浪纹样的茶杯；需要双手工作时可放在旁边、挂在包上或由小水母托住。

可选辅助识别点：深蓝内搭、贝壳斜挎包、透明浅蓝小水母伙伴、水滴或气泡符号。

## 正文配图中的简化方式

角色设定图较精细，正文配图必须主动简化：

- 使用 2.5–3.5 头身的轻 Q 版比例。
- 用细黑线或深海军蓝线勾勒，局部平涂，禁止厚重赛璐璐和精细光影。
- 每次只保留 3–4 个外形锚点，不把所有挂饰、纹样、包袋细节都画出来。
- 海灵通常占画面 12%–25%；角色状态图最多约 35%。
- 表情明亮、专注、有行动感；避免只靠卖萌。

## 核心职责与动作池

让海灵承担概念动作，例如：

- 拉绳、接线、搭桥、开闸、修补断点。
- 掌舵、冲浪、潜水、打捞、放锚、点亮灯塔。
- 分类、筛选、搬运、贴标签、旋转阀门、推动齿轮。
- 倒茶、过滤茶叶、观察茶汤沉淀、用蒸汽传递信号。
- 把混乱信息装进瓶子、把节点连成航线、把结果送到岸边。
- 鼓励犹豫的角色迈出一步，但不替代内容本身。

动作必须服务核心意思。喝茶可以作为节奏和温度，不应成为每张图唯一动作。

## 小水母伙伴

小水母是可选的二级角色，适合表示提示、反馈、通知、自动化助手或轻量状态。它不能代替海灵承担主动作，也不要每张都出现。

## 禁止

- 不要把海灵替换成动物、黑色抽象生物或通用吉祥物。
- 不要画成写实人物、成熟性感角色或完整动漫海报主角。
- 不要复制任何已有动漫角色的发型、服装和标志性姿势。
- 不要使用复杂盔甲、繁复裙装、巨量挂饰、闪光背景和大眼特写。
- 不要让她只站在角落挥手、比心、喝茶或充当讲解贴纸。
- 不要让角色面积与细节压过知识结构。

## 一致性判断

读者应在一秒内通过“蓝色浪发 + 橙色海星 + 白蓝斗篷 + 茶杯”识别海灵，同时能看出她正在推动这张图的核心动作。

## 最高优先级三视图参考

生成海灵时优先使用以下两张三视图，而不是依赖文字自由发挥：

- `assets/character/hailing-normal-three-view.png`：常规比例正面、侧面、背面三视图。用于锁定完整服装结构、白蓝斗篷、波浪纹、单肩背包、鞋子、后背发束和背面配件。
- `assets/character/hailing-chibi-three-view.png`：Q 版正面、侧面、背面三视图。用于锁定正文配图默认角色比例、头身比、简化线条和可爱程度。

任何生成或改图任务都必须把海灵当成同一个固定人物来处理。不能只保留“蓝发、海洋、茶杯”等主题元素后重新设计角色。三视图里的发型轮廓、海星与贝壳发饰、白蓝斗篷、深蓝内搭、单肩包、鞋子和背面轮廓是身份锚点。
