# Shot List 模板

默认输出 4–8 张。每张严格只选一个认知锚点。

```markdown
## 配图策略

### 01. {图名}
- 放置位置：{哪一段之后 / 哪个小节}
- 核心意思：{一句话}
- 结构类型：{Workflow / 系统局部 / 前后对比 / 角色状态 / 概念隐喻 / 方法分层 / 地图路线 / 小漫画分镜}
- 海灵动作：{必须是具体动词}
- 主隐喻：{一个新鲜的低科技物理隐喻}
- 主物件：{1–2 个}
- 建议标注：{3–6 个短词}
- 颜色重点：{海洋蓝 / 浅青 / 珊瑚橙中的必要用法}
- 生成风险：{文字过多 / 角色过细 / 太像 PPT / 海洋装饰过量等}

### 02. {图名}
...
```

## 规划规则

- 不要为了凑数量给每一段配图。
- 相邻图片不要重复结构、主物件或海灵动作。
- 至少一张使用非海洋物件，避免主题元素单调重复。
- 先写清楚核心意思，再设计角色动作。
- 标注词必须能直接放进图片，避免完整句子。
- 只有用户明确要求“一图看懂”时，才规划教程总览页。
