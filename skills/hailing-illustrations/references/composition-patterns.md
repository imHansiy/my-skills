# 构图模式与原创规则

## 基础结构类型

一次只选一种主结构。教程总览可以把同一主题的 2–4 个连续动作组合起来，但不要混合多个框架。

### Workflow 流程

适合：输入 → 处理 → 输出、自动化、内容生产、业务链路。

画法：信息从一侧进入，海灵在中间掌舵、过滤、接线或推动装置，结果从另一侧出去。用少量蓝色水流或珊瑚橙路径表达主方向。

### 系统局部

适合：信息来源、过滤器、数据库、模块协作、agent 系统局部。

画法：只画 3–5 个核心模块。海灵正在连接其中最关键的一处，而不是站在系统旁边讲解。

### 前后对比

适合：混乱/有序、手动/自动、分散/收拢、犹豫/行动。

画法：两侧或上下对比，中间保留一个明确转折动作。海灵可以把散乱漂流物收进航道，或把打结绳索解成一条线。

### 角色状态

适合：用户痛点、工具焦虑、卡住到跑起来、学习过程。

画法：2–4 个小状态，每格一个动作和一个短标注。角色变化服务内容，不做表情包合集。

### 概念隐喻

适合：信息仓库、决策黑盒、信任积累、内容工厂、反馈闭环。

画法：一个有记忆点的低科技物件或海上装置，少量输入，一个明确输出。海灵必须操作它。

### 方法分层

适合：方法论框架、能力栈、系统层级。

画法：抽屉、船舱、码头平台、茶盒或逐层容器；不要正式金字塔。海灵在搭建、搬运或检查其中一层。

### 地图路线

适合：学习路线、用户旅程、从想法到上线、内容承接路径。

画法：一条弯曲但清楚的航线，3–5 个节点。海灵掌舵、点灯或拉起浮标。

### 小漫画分镜

适合：失败到成功、使用前后、真实过程、轻吐槽、短教程总览。

画法：2–4 个稀疏小场景，每格只表达一个动作。分隔线要轻，不要做成课程卡片墙。

## 原创隐喻生成法

每次从当前内容重新发明隐喻：

1. 把抽象概念换成物理动作：卡住、漏掉、打结、漂散、汇流、过滤、沉淀、点亮、靠岸、回流、发酵、升温。
2. 把结构换成一个低科技物件：绳索、闸门、抽屉、筛网、旧机器、邮筒、瓶子、灯塔、浮标、船舱、茶壶、滤茶器。
3. 让海灵承担动作：拉、推、接、捞、筛、拧、守、修、标记、掌舵、放锚、倒茶、点灯。
4. 最后才决定是否加入一个海洋或茶的回声。它必须帮助理解，不能只是装饰。

## 物件池

每张只选 1–2 个主物件：

- 通用：纸箱、抽屉、绳结、漏斗、秤、门、桥、梯子、水管、转盘、黑盒、打孔器、旧工位。
- 海洋：浮标、灯塔、小船、锚、渔网、航线、码头、玻璃瓶、贝壳盒、潮汐闸门。
- 茶系：茶壶、茶杯、滤网、茶叶、蒸汽、茶盘、茶罐、沉淀层。

## 海灵动作池

- 连接：拉绳、接线、搭桥、系结、开闸。
- 处理：筛选、过滤、分拣、称重、标记、装瓶。
- 推动：掌舵、冲浪、推车、转阀、点灯、放锚。
- 修复：补洞、捞回、解结、扶梯、重新铺路。
- 茶系：倒茶、滤茶、观察沉淀、用蒸汽传递信号。

动作要服务核心意思，不要为了海洋感而强行冲浪，也不要每张都端茶杯。

## 反复刻规则

- 默认不要打开 `assets/examples/`；只有出现风格漂移时才低频校准。
- 不要复刻示例中的 Odoo 文案、四格位置、图标排序、角色姿势或边框。
- 不要连续多张都使用浪花、灯塔、茶杯或小船。
- 同类主题必须换主动作和主物件。例如“信息汇聚”可以是收网、装瓶、系浮标、滤茶或码头靠岸，不要固定成水流汇聚。
- 先保证隐喻成立，再添加海灵身份符号。
